Maven构建工具

Genlg 2019-09-17 maven
# Maven存在的理由

  • 项目特别庞大时，仅依靠package划分模块，十分繁杂，不利于团队分工，Maven可以将一个项目拆分程多个单独的项目

  • 项目中的jar包重复出现，工程文件冗余，不便于管理，Maven可以将jar保存到仓库中，每个项目仅需要引用jar的地址即可

  • 每个项目的jar包需要依赖其他的jar包，需要手动加入依赖的jar包，下载来源多，不统一，Maven可以从指定的仓库中以规范的方式自动导入jar包依赖的jar包

# Maven是什么？

  • 服务java平台的自动化构建工具（gradle比maven更加先进）

  • 构建过程

    概念：以java源文件，配置文件，jsp等资源去生产一个可以运行的项目

    1. 编译：.java ----编译----->.class -->加入到jvm内存
    2. 部署
    3. 搭建

  • 构建的环节（生命周期）

    1. 清理：将以前编译的class文件删除，为下一次编译做准备
    2. 编译：将java文件编译成字节码文件
    3. 测试：自动测试，自动调用junit测试程序
    4. 报告：测试程序执行的结果
    5. 打包：动态web工程打成war或jar包
    6. 安装：将打包的文件存放到指定的仓库的位置
    7. 部署：将war包或jar包复制到servlet容器中，使其能够运行

  • Maven核心概念

    1. 约定的目录结构(约定>配置>编码)
    2. pom.xml文件
    3. 坐标
    4. 依赖
    5. 仓库
    6. 生命周期
    7. 继承
    8. 聚合

  • Maven插件

maven核心程序仅定义了抽象的生命周期，具体的执行操作需要其他插件来完成，他会在本地仓库(默认仓库在系统盘中的.m2文件目录中)，如果本地仓库没有，则会通过联网的方式下载执行命令需要的插件。

可以通过修改maven核心程序中的corf/setting.xml文件中的localRespository标签指定maven插件下载存放的文件目录位置

  • POM

    Project Object Model 项目对象模型，类似DOM 文档对象模型

    pom.xml文件是maven中的核心配置文件

    • 坐标

      • groupid：公司/组织+项目名
      • artifactid：模块名
      • version：版本
