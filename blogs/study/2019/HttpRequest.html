# JSON

json是一种轻量型的数据交换格式，完全独立于编程语言的文本格式来存储和表现数据。

# JSON的对象结构

{

​ key1 : value1, //键值对

​ key2 : value2, // key为字符串 ，value可以为字符串，整型，浮点型，数组

​ ... // key代表对象的属性， value 代表对象属性的值

​ keyn: valuen

}

# JSON的生成

# 1.需要json的jar包的支持
包名
jackson-core-2.8.8.jar
jackson-annotations-2.8.8.jar
jackson-databind-2.8.8.jar
# 2.JSON的方法
JSON.方法名 功能
JSON.stringify(json对象) 将json对象转换成json字符串，返回json字符串对象
JSON.parse(json字符串) 将json字符串转换成json对象，返回json对象

# Ajax的格式

$.ajax({
    type : "post",    
    url : "url",      //向后台服务器请求的路径
    contentType : "application/json;charset=UTF-8",//请求的数据格式
    dataType : "json",//服务器响应的数据格式，可以省略不写，浏览器自动识别
    data : {},      //发送的数据   json字符串或者json对象
    beforeSend : function() {
        //发送数据到服务器之前等待中的操作
    },
    success : function(result) {
        //请求成功，获得返回的json对象
    },
    error : function() {
        //请求服务器数据失败
    }
});

# Content-Type

Content-Type(编码类型) 作用
application/x-www-form-urlencoded(默认) 表单数据被编码为名称/值对
text/plain 窗体数据以纯文本形式进行编码,不含任何控件或格式字符
multipart/form-data 二进制数据
application/json json数据

表单中当action为get时候，浏览器用x-www-form-urlencoded的编码方式把form数据转换成一个字串（name1=value1&name2=value2...），然后把这个字串append到url后面，用?分割，加载这个新的url。当action为post时候，浏览器把form数据封装到http body中，然后发送到server。如果没有 type=file 的控件，用默认的 application/x-www-form-urlencoded 即可。

# @RequestBody

将请求体中的数据绑定到方法的形参中

application/json格式的数据需要@RequestBody注解来绑定，在SpringMVC环境中,@RequestBody接收的是一个json对象的字符串，而不是一个json对象，因此，@RequestBody需要接的参数是一个String化的json，前台js代码可以直接使用JSON.stringify(json)这个方法来转化。从@RequestBody名称上来看，也就是说要读取的数据在请求体（body）里，所以要发post请求。@RequestBody接收的是一个String的Json，那么要设置请求contentType:"application/json，明确的告诉服务器发送的内容是json。

# @ResponseBody

方法的返回结果直接写入 HTTP response body 中，异步获取 json 数据，加上 @ResponseBody 后，会直接返回 json 数据。

# 媒体类型

媒体类型 含义
text/html HTML格式
text/plain 纯文本格式
text/xml, application/xml XML数据格式
application/json JSON数据格式
image/gif gif图片格式
image/png png图片格式
application/octet-stream 二进制流数据
application/ x-www-form-urlencoded form表单数据
multipart/form-data 含文件的form表单
