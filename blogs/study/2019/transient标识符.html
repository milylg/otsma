transient标识符

Gen 2019-06-03 transient

被这个标识符修饰的变量不参与类的序列化过程，字段的生命周期仅存于调用者的内存中而不会写到磁盘里持久化。

代码清单 - 实现Serializable接口的序列化类 
public class Serializedemo implements Serializable{

	private static final long serialVersionUID = -1624564379892526699L;
	
	private transient String value; //transient修饰的变量不参与序列化
	
	public static String str;
	
	private String string;
	
	public Serializedemo() {
		value = "hhhhhh";
		string = "jjjjjj";
	}

	@Override
	public String toString() {
		return "Serializedemo [value=" + value + ", string=" + string + "]";
	}
}

代码清单 - 实现Externalizable接口的序列化类 
public class Externalizedemo implements Externalizable {
	
	private transient String value;
	
	public static String str;
	
	private String string;
	
	public Externalizedemo() {
		value = "bbbbbbbbbbb";
		string = "cccccccccccc";
		str = "dddddddddddd";
	}

	/******************************************************************************** 
	 * CoverMethodName: writeExternal
	 * Description:保存其内容，它可以通过调用 DataOutput 的方法来保存其基本值
	 * 或调用 ObjectOutput 的 writeObject 方法来保存对象、字符串和数组。
	 * CoverTime:2019年6月3日下午4:04:24
	 * @param out
	 * @throws IOException
	 ********************************************************************************/
	@Override
	public void writeExternal(ObjectOutput out) throws IOException {
		out.writeObject(value);
		out.writeObject(string);
	}

	/******************************************************************************** 
	 * CoverMethodName: readExternal
	 * Description: 恢复其内容，它通过调用 DataInput 的方法来恢复其基础类型
	 * 调用 readObject 来恢复对象、字符串和数组。
	 * CoverTime:2019年6月3日下午4:04:24
	 * @param in
	 * @throws IOException
	 * @throws ClassNotFoundException
	 ********************************************************************************/
	@Override
	public void readExternal(ObjectInput in) throws IOException, ClassNotFoundException {
		//顺序读取对象
		value = (String) in.readObject();
		string = (String) in.readObject();
	}

	@Override
	public String toString() {
		return "Externalizedemo [value=" + value + ", string=" + string + "]";
	}
}

运行结果

Externalizedemo [value=bbbbbbbbbbb, string=cccccccccccc]

对象的序列化可以通过实现两种接口来实现，若实现的是Serializable接口，则所有的序列化将会自动进行，若实现的是Externalizable接口，则没有任何东西可以自动序列化，需要在writeExternal方法中进行手工指定所要序列化的变量，这与是否被transient修饰无关。因此第二个案例输出的是变量value初始化的内容，而不是null。

注意

一个静态变量不管是否被transient修饰，均不能被序列化。测试中的静态变量也可以输出在控制台上，但是这个值是JVM中的值，并不是序列化文件中读出来的值。
