罗马数字转整数

Gen 2020-10-21 剑指offer

# 题目描述

罗马数字包含以下七种字符: I， V， X， L，C，D 和 M。
字符 数值
I 1
V 5
X 10
L 50
C 100
D 500
M 1000

例如， 罗马数字 2 写做 II ，即为两个并列的 1。12 写做 XII ，即为 X + II 。 27 写做 XXVII, 即为 XX + V + II 。

通常情况下，罗马数字中小的数字在大的数字的右边。但也存在特例，例如 4 不写做 IIII，而是 IV。数字 1 在数字 5 的左边，所表示的数等于大数 5 减小数 1 得到的数值 4 。同样地，数字 9 表示为 IX。这个特殊的规则只适用于以下六种情况：

  • I 可以放在 V (5) 和 X (10) 的左边，来表示 4 和 9。

  • X 可以放在 L (50) 和 C (100) 的左边，来表示 40 和 90。

  • C 可以放在 D (500) 和 M (1000) 的左边，来表示 400 和 900。

  • 给定一个罗马数字，将其转换成整数。输入确保在 1 到 3999 的范围内。

# 解决思路

  • 按照题目意思编写逻辑
  • 分析功能需求，发现整体的计算规律

# 代码实现

第一种：（自己实现的烂代码）

public int romanToInt(String s) {
        int sum = 0;
        int len = s.length();
        for (int i = 0; i < len; i ++) {
            char c = s.charAt(i);
            char r = ' ';
            if (i < len - 1) {
                r = s.charAt(i + 1);
            }
            if (c == 'I') {
                if (r == 'V' || r == 'X') {
                    sum += getValueBy(r) - getValueBy(c);
                    i ++;
                } else {
                    sum += getValueBy(c);
                }
            } else if (c == 'X') {
                if (r == 'L' || r == 'C') {
                    sum += getValueBy(r) - getValueBy(c);
                    i ++;
                } else {
                    sum += getValueBy(c);
                }
            } else if (c == 'C') {
                if (r == 'D' || r == 'M') {
                    sum += getValueBy(r) - getValueBy(c);
                    i ++;
                } else {
                    sum += getValueBy(c);
                }
            } else {
                sum += getValueBy(c);
            }
        }
        return sum;
    }

上述的代码仅仅只是对规则描述进行翻译的代码，并没有优化，一般写完代码，可以尝试对代码结构和设计逻辑进行优化，这是可以进步的关键点。

优化后的代码清单

    public int romanToInt(String s) {
        int len = s.length();
        int sum = 0;
        for (int cur = 0; cur < len - 1; cur ++) {
            int nexv = getValueBy(s.charAt(cur + 1)); // 这里调用了两次 getValueBy函数
            int curv = getValueBy(s.charAt(cur));
            if (nexv > curv) {
                sum -= curv;
            } else {
                sum += curv;
            }
        }
        return sum + getValueBy(s.charAt(len - 1));
    }

再次优化：优化的地方是nexv和curv的值存在递推的关系，因此只要通过一次计算后置节点的值，再循环结束之前将后置节点赋值给前置节点，这样避免了调用两次getValueBy(s.charAt(cur))操作。

public int romanToInt(String s) {
    int len = s.length();
    int sum = 0;
    int curv = getValueBy(s.charAt(0));
    for (int cur = 1; cur < len; cur ++) {
        int nexv = getValueBy(s.charAt(cur)); // 现在只调用了一次函数
        if (nexv > curv) {
            sum -= curv;
        } else {
            sum += curv;
        }
        curv = nexv;
    }
    return sum + getValueBy(s.charAt(len - 1));
}

调用的公共私有函数

    private int getValueBy(char c) {
        switch (c) {
            case 'I': return 1;
            case 'V': return 5;
            case 'X': return 10;
            case 'L': return 50;
            case 'C': return 100;
            case 'D': return 500;
            case 'M': return 1000;
            default : break;
        }
        return  0;
    }
