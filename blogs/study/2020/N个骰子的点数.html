n个骰子的点数

Gen 2020-10-12 剑指offer

# 题目描述

题目：把n个骰子扔在地上，所有骰子朝上一面的点数之和为s。

输入n，打印出s的所有可能的值出现的概率。

# 抽象建模能力

  • 建模的第一步是选择合理的数据结构来表述问题
  • 建模的第二步是分析模型中的内在规律，并用编程语言表述这种规律

在建模的过程中除了需要具备扎实的数学基础和编程能力之外，还需要具有敏锐的洞察力和丰富的想象力。

# 基于递归求骰子点数

使用递归的方式很简单，将点数和点数出现的次数存储在一个map映射中，它记录了n个色子出现sum点数的次数，结束条件是第n个色子摇完，在结束时进行统计，如果该点数已经存在，从中获取出现的次数，并加上当前出现的一次，再更新映射。每摇一颗色子，它的点数有6种，因此每次摇色子的时候，需要计算它的6种可能。

# 代码清单 - 递归函数解决色子点数出现的次数

private void function(int n, int sum, Map<Integer, Integer> map) {
    if (n <= 0) {
        if (map.containsKey(sum)) {
            Integer count = map.get(sum);
            map.put(sum, count + 1);
            return;
        }
        map.put(sum, 1);
        return;
    }
    // 当前摇色子所有点数的可能【1-6】，每种可能都需要参加计算总点数
    for (int i = 1; i <= 6; i ++) {
        function(n - 1, sum + i, map);
    }
}

# 基于循环求骰子点数

在一次循环中，第一个数组中的第n个数字表示骰子和为n出现的次数。在下一循环中，我们加上一个新的骰子，此时和为n的骰子出现的次数应该等于上一次循环中骰子点数和为n－1、n－2、n－3、n－4、n－5与n－6的次数的总和，所以我们把另一个数组的第n个数字设为前一个数组对应的第n－1、n－2、n－3、n－4、n－5与n－6之和。

------ 参考《剑指Offer》

通过画图可知以下规律

投 N 个色子，它的总点数可能值有 5 * N + 1 个

因此，需要创建计数数组SUM的空间为 5 * N + 1;

它的可能总点数的范围为[N, 6N]

初始化N = 1时，它的点数出现在 [1, 6]中

第一次投色子，它的结果是每种点数出现的次数为 [1, 1, 1, 1, 1, 1]。

当第二次投色子时，如何计算点数的可能值的次数？

此时，已知可能结果有11个，点数值范围在[2, 12]

下图是第二次投掷色子的结果

从下面的图，不难看出，第二次投掷色子，如果想二次投掷出6个点数，已知第二次投掷的点数，结果取决与它的第一次投掷的是几个点数

第二次投掷点数可能为1，那么第一次需要投掷点数为5，第一次投掷点数为5出现次数为1

第二次投掷点数可能为2，那么第一次需要投掷点数为4，第一次投掷点数为5出现次数为1

第二次投掷点数可能为3，那么第一次需要投掷点数为3，第一次投掷点数为5出现次数为1

第二次投掷点数可能为4，那么第一次需要投掷点数为2，第一次投掷点数为5出现次数为1

第二次投掷点数可能为5，那么第一次需要投掷点数为1，第一次投掷点数为5出现次数为1

因为二次投掷点数和为6，故第二次投掷为6点的次数不计入，不符合条件

共计5种可能出现二次投掷出6个点数

这里讨论的是当前投掷的点数小于等于6种的情况

接下来讨论当前投掷的点数的可能等于6种的情况

通过画图演示流程，可知与上述讨论情况一致，投掷出9点的次数是临时数组Temp中值的和。

例如，我要计算总点数是9时，一共有几种可能性。

第二次投掷色子的点数有6种可能，分别是1，2，3，4，5，6

那么第一次投掷色子的点数需要满足在[3, 8]范围

把第一次投掷色子的点数范围在[3, 8]的出现次数相加，即第二次投掷出点数为9的出现次数

临时数组的作用是为了记录前一个色子在当前和前五次投掷的次数。

此时临时数组记录了第一次投掷色子的点数范围在[3, 8]的出现次数，结果是4次。

游标的作用是将上一个色子的计数数组SUM的下一个索引位置的值赋值到游标指向的索引位置，用于覆盖掉滑动区间中最左边的值。

# 代码清单 - 获取点数出现的次数

public int[] getProbability(int n) {
    int[] c = new int[5 * n + 1];
    // initialized
    c = init(c, 1, 6);
    int[] temp = new int[6];
    // recurrence
    int cursor = 0;
    for (int k = 2; k <= n; k ++) {
        int arrayMaxIndex = 5 * k;
        for (int j = 0; j <= arrayMaxIndex; j ++) {
            if (cursor == 6) {
                cursor = 0;
            }
            temp[cursor ++] = c[j];
            c[j] = sum(temp);
        }
        temp = init(temp, 0, 6);
        cursor = 0;
    }
    return c;
}

// 计算临时数组的和
private int sum(int[] array) {
    int sum = 0;
    for (int i : array) {
        sum += i;
    }
    return sum;
}

// 初始化临时数组的值，用于下一个骰子的投掷计算
private int[] init(int[] array, int value, int length) {
    for (int i = 0; i < length; i++) {
        array[i] = value;
    }
    return array;
}

# 总结与反思

这里的代码与剑指Offer不同，但是思想是一样的，只是具体的实现方式不一样。

  • 滑动区间和游标的作用
  • 递归分解问题：自顶向下分解问题
  • 递推公式，找到分解问题的粒度，比如这里按骰子的个数，自底而上的计算得出每个点数出现的次数
