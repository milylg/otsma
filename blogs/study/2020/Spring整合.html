# SpringMVC

Spring整合SpringMVC框架 [XML+注解]

目前开发完全使用xml配置不太频繁，一般结合注解和xml使用。使用xml配置Spring的流程大概有下列几个步骤：

  • 开启注解处理器<context:annotation-config/>，当使用@Conponent注解时需要。
  • 创建java类，并用@Conponent，@Repository，@Service，@Controller标注类属于哪种分层。
  • 在xml中使用<bean></bean>描述javabean的类信息，包括类全限定名，成员属性，构造方法等。这一步其实可以使用<context:component-scan base-package="packet name"/>开启注解扫描，它会让Spring容器在指定包的范围内扫描有上述注解的类，并注入到容器中。在使用注解扫描类时，类中依赖的成员属性需要使用注入的注解，例如@Autowired，@Qualifier，或@Resource。

这些操作是Spring如何将对象放入到容器的操作。其中涉及到两个概念：

  • IOC [控制反转] 在没有Spring容器的时候，我们创建对象是通过new的方式，这样造成程序耦合性过高，不利于扩展，现在通过Spring框架来管理Spring创建的对象，我们仅仅需要定义类即可，对象交由Spring容器创建和管理，Spring容器管理了对象的整个生命周期。通俗的说，IOC容器控制了对象对外部资源（对象和文件）的获取，这是反转的部分，这部分的职责本来是由程序员来处理的，现在将这个职责交给Spring容器去做，它自动注入对象需要的依赖。IOC容器的实现原理其实是工厂模式+注解（xml）+反射结合实现的，对象其实都存在对象工厂中。这是一种设计思想，可以指导如何去设计一个低耦合的系统。

  • DI [依赖注入] 应用程序依赖于Spring容器，为应用程序提供对象服务，容器通过外部资源注入创建的对象所需的依赖资源，包括对象或文件。概念其实和IOC差不多，只是站的角度不同，DI是站在Spring容器的角度，而IOC是站在程序员的角度看待对象的创建过程。

springmvc框架使用了mvc的设计模式对表现层的处理流程进行了职责分明的设计。其中的模型，视图，控制器概念是这个框架的核心，并非对应我们的model（entry），controller这些类，这是一个概念上的混淆。下图是表现层的mvc设计模式

与MVC框架不同，mvc设计模式解决的问题是将视图和数据分离管理，降低耦合度。设计模式和框架的区别在于：设计模式是解决不同环境中反复出现的问题对应相似的解决方案，注重设计重用，抽象程度大于框架。而框架是为解决某一个特定领域的问题而产生的，注重代码重用，一个框架中运用了多种设计模式。用一个比喻来说明它们之间的区别：设计模式是建筑图纸好比框架是建筑物实体。

还有一个混淆的概念，MVC架构，首先在抽象程度上，架构介于模式和框架之间，既关注代码重用，也关注设计重用。在实现层面上，它的目的是分配职责，采用三层架构分层：表现层、业务层、持久层。表示层负责接收用户请求、转发请求、显示数据等；业务层负责组织业务逻辑；持久层负责持久化业务对象。而SpringMVC框架提供的是表现层的解决方案。

图解来源网络，流程比较清晰

spring mvc是spring框架的一个模块，springmvc和spring无需通过中间整合层进行整合，它大概需要下列配置流程：

  • 配置web层的扫描范围
  • 开启mvc注解驱动<mvc:annotation-driven />
  • 配置静态资源访问映射
  • 配置视图解析器
  • 配置URL请求拦截器

如果需要更加复杂的功能需要参考官方文档配置需要的功能。

<beans xmlns="http://www.springframework.org/schema/beans"
    xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" 
    xmlns:mvc="http://www.springframework.org/schema/mvc"
    xmlns:context="http://www.springframework.org/schema/context"
    xsi:schemaLocation="http://www.springframework.org/schema/beans 
    http://www.springframework.org/schema/beans/spring-beans-4.3.xsd 
    http://www.springframework.org/schema/mvc 
    http://www.springframework.org/schema/mvc/spring-mvc-4.3.xsd 
    http://www.springframework.org/schema/context 
    http://www.springframework.org/schema/context/spring-context-4.3.xsd">
    <!-- 加载属性文件 -->
    <!--<context:property-placeholder location="classpath:resource.properties" />-->
    <!-- 配置扫描器 -->
    <context:component-scan  base-package="org.example.web" />
    <!-- 注解驱动：配置处理器映射器和适配器 -->
    <mvc:annotation-driven />
    <!--配置静态资源的访问映射，此配置中的文件，将不被前端控制器拦截 -->
    <mvc:resources location="/js/" mapping="/js/**" />
    <mvc:resources location="/css/" mapping="/css/**" />
    <mvc:resources location="/fonts/" mapping="/fonts/**" />
    <mvc:resources location="/images/" mapping="/images/**" />	
    <!-- 配置视图解释器ViewResolver -->
    <bean id="jspViewResolver" class=
    "org.springframework.web.servlet.view.InternalResourceViewResolver">
		<property name="prefix" value="/WEB-INF/jsp/" />
		<property name="suffix" value=".jsp" />
    </bean>	
    
    <!-- 配置拦截器 -->
	<mvc:interceptors>
    	<mvc:interceptor>
        	<mvc:mapping path="/**" />
        	<bean class="org.example.interceptor.LoginInterceptor" />
    	</mvc:interceptor>
	</mvc:interceptors>	
</beans>

如果项目中不通过web.xml配置DispatcherServlet，那么需要通过javaconfig类和AbstractAnnotation-ConfigDispatcherServletInitializer实现类来引导加载web项目。

需要注意的是web层的请求处理类需要加注@RestController注解，@RestController与@Contorller的区别是前者支持RESTful风格的接口，且不需要在方法上标注@ReponseBody。

web.xml配置如下：

<?xml version="1.0" encoding="UTF-8"?>
<web-app xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" 
xmlns="http://java.sun.com/xml/ns/javaee" 
xsi:schemaLocation="http://java.sun.com/xml/ns/javaee 
					http://java.sun.com/xml/ns/javaee/web-app_3_0.xsd" 
					id="WebApp_ID" version="3.0">
  <!-- 配置前端控制器 -->				
  <servlet>
    <servlet-name>springmvc</servlet-name>
    <servlet-class>
              org.springframework.web.servlet.DispatcherServlet
    </servlet-class>
    <init-param>
      <param-name>contextConfigLocation</param-name>
      <param-value>classpath:springmvc-config.xml</param-value>
    </init-param>
    <load-on-startup>1</load-on-startup>
  </servlet>
  <servlet-mapping>
    <servlet-name>springmvc</servlet-name>
    <url-pattern>/</url-pattern>
  </servlet-mapping>
  
  	<!-- 配置编码过滤器 -->
	<filter>
		<filter-name>CharacterEncodingFilter</filter-name>		
		<filter-class>
       		org.springframework.web.filter.CharacterEncodingFilter
		</filter-class>
		<init-param>
    		<param-name>encoding</param-name>
       		<param-value>UTF-8</param-value>
		</init-param>
	</filter>
	<filter-mapping>
   		<filter-name>CharacterEncodingFilter</filter-name>
		<url-pattern>/*</url-pattern>
	</filter-mapping>
</web-app>

Spring整合SpringMVC框架 [JavaConfig+注解]

使用配置类向spring容器中注入springmvc需要的对象以及一些设置，也可以使用@ImportResource的注解导入xml形式的springmvc配置文件。@EnableWebMvc开启mvc的注解驱动，相当于<mvc:annotation-driven>。使用@ComponentScan(basePackages = "org.example.web")注解扫描请求处理器，这部分对象归springmvc管理，与spring无关。需要注意的是spirng开启的包扫描不能包括这个用于请求处理的包。通过使用@Configuration来表明当前类是一个配置类，@Bean标注的方法是进行bean的初始化并注入到spring容器中。

程序清单：SpringmvcConfigurator.java

@EnableWebMvc
@ComponentScan(basePackages = "org.example.web")
@Configuration
public class SpringmvcConfigurator extends WebMvcConfigurationSupport {

    /**
     * 配置类继承的类可以选择下列其中一个
     * - WebMvcConfigurationSupport
     * - WebMvcConfigurerAdapter
     * 他们之间的区别？
     * 在类的构造上
     * WebMvcConfigurerAdapter 全部是适配的接口
     * WebMvcConfigurationSupport 已经注入需要的类，将对象的自定义配置交由它的子类来设置
     */

    /**
     * @return 视图解析器
     */
    @Bean
    public ViewResolver createViewResolver() {
        InternalResourceViewResolver resolver = new InternalResourceViewResolver();

        return resolver;
    }

    /**
     * 可以重写WebMvcConfigurationSupport父类中对子类开放的方法用于功能自定义
     *
     * - void addInterceptors(InterceptorRegistry registry) 对拦截器进行设置
     *
     * - void addResourceHandlers(ResourceHandlerRegistry registry) 对静态资源进行设置
     *
     * - void addFormatters(FormatterRegistry registry) 格式转换器设置
     *
     * - void addViewControllers(ViewControllerRegistry registry) 视图控制器
     *
     * - void configureViewResolvers(ViewResolverRegistry registry) 配置视图解析器
     * 
     * etc......
     */
    
    /**
     * 自定义springmvc组件的属性
     */
    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {
        registry.addResourceHandler("/static/**").addResourceLocations("/static/");
    }
}

web项目的启动需要配置类的引导，需要实现AbstractAnnotationConfigDispatcherServletInitializer的子类进行配置引导。

程序清单：WebInitializer.java

public class WebInitializer extends AbstractAnnotationConfigDispatcherServletInitializer {
    /**
     *  注册顺序->listener->filter(Shiro安全框架.etc)->servlet(SpringMVC)..
     *  必须按照顺序注册,所以必须重写onStarup
     */
    @Override
    public void onStartup(ServletContext servletContext) throws ServletException {

        registerContextLoaderListener(servletContext);
        // 在这里注册过滤器

        registerDispatcherServlet(servletContext);
    }

    /**
     * 注册过滤器解决post乱码问题
     */
    @Override
    protected Filter[] getServletFilters() {
        CharacterEncodingFilter filter=new CharacterEncodingFilter();
        filter.setEncoding("utf-8");
        filter.setForceEncoding(true);
        return new Filter[] {filter};
    }


    /**
     * 此方法负责加载Service和其他第三方包的初始化配置如service层和DataAccessObject层的框架和类
     * <p>
     * 类似 application-config.xml
     *
     * AppRootConfig.class,AppShiroConfig.class,AppRedisConfig.class
     */
    @Override
    protected Class<?>[] getRootConfigClasses() {
        return new Class[] { SpringConfigurator.class };
    }

    /**
     * springmvc表现层的配置
     *
     * 引导web项目加载，指定Spring容器的配置类为SpringConfig.class，
     *
     * 注意WebInit本身并不是Spring容器的组件，
     *
     * 而是通过继承AbstractAnnotationConfigDispatcherServletInitializer实现引导容器加载的作用
     *
     * 这里写入用以取代Spring和SpringMVC的配置文件
     */
    @Override
    protected Class<?>[] getServletConfigClasses() {
        return new Class[]{ SpringmvcConfigurator.class };
    }

    /**
     * springmvc 的请求路径映射
     *
     * 这里配置默认前端控制器.这里所有匹配*.do的访问都会被捕捉到.
     * 也可以配置为"/" 不会拦截.jsp
     * /* 全部拦截
     */
    @Override
    protected String[] getServletMappings() {
        return new String[]{"/"};
    }
}

<?xml version="1.0" encoding="UTF-8" ?>
<!DOCTYPE configuration PUBLIC "-//mybatis.org//DTD Config 3.0//EN"
        "http://mybatis.org/dtd/mybatis-3-config.dtd">
<configuration>
    <!-- 通过别名简化对类的使用 -->
    <typeAliases>
        <!--<typeAlias type="cn.itcast.scm.entity.Dept" alias="Dept" /> -->

            mybatis会自动扫描你指定包下面的javabean,
            并且默认设置一个别名，默认的名字为非限定类名来作为它的别名。-->
        <package name="org.example.bean"/>
    </typeAliases>
    <!--接口和xml文件映射配置-->
    <mappers>
        <package name="mappers"/>
<!--        <mapper resource="mappers/UserRepository.xml" />-->
    </mappers>
</configuration>

对数据源和事务切面的配置

<?xml version="1.0" encoding="UTF-8"?>
<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance" xmlns:tx="http://www.springframework.org/schema/tx"
       xmlns:aop="http://www.springframework.org/schema/aop" xmlns:context="http://www.springframework.org/schema/context"
       xmlns:p="http://www.springframework.org/schema/p"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
    http://www.springframework.org/schema/beans/spring-beans-3.2.xsd
    http://www.springframework.org/schema/context
    http://www.springframework.org/schema/context/spring-context-3.2.xsd
    http://www.springframework.org/schema/tx
    http://www.springframework.org/schema/tx/spring-tx-3.2.xsd
    http://www.springframework.org/schema/aop http://www.springframework.org/schema/aop/spring-aop-3.2.xsd">

    <context:property-placeholder location="classpath:jdbc.properties"/>

    <!-- 配置数据源，记得去掉myBatis-config.xml的数据源相关配置 -->
    <bean id="dataSource" class="com.alibaba.druid.pool.DruidDataSource">
        <property name="driverClassName" value="${driverClassName}" />
        <property name="url" value="${url}" />
        <property name="username" value="${username}" />
        <property name="password" value="${password}" />
    </bean>
    <!-- 配置session工厂 -->
    <bean id="sqlSessionFactory" class="org.mybatis.spring.SqlSessionFactoryBean">
        <property name="dataSource" ref="dataSource" />
        <property name="configLocation" value="classpath:mybatis-config.xml" />
        <!--配置扫描式加载SQL映射文件,记得去掉mybatis-config配置-->
        <!--<property name="mapperLocations"    
                      value="classpath:org.example.repository/*.xml"/>-->
    </bean>

    <!-- 配置事务管理器,管理数据源事务处理 -->
    <bean id="transactionManager"
          class="org.springframework.jdbc.datasource.DataSourceTransactionManager">
        <property name="dataSource" ref="dataSource" />
    </bean>
    <!-- 配置事务通知 标签作用是创建一个TransactionInterceptor-->
    <tx:advice id="advice" transaction-manager="transactionManager">
        <tx:attributes>
            <!-- 默认只处理运行时异常，可加rollback-for="Exception/Throwable"等处理所有异常或包括错误 -->
            <tx:method name="insert*" propagation="REQUIRED"
                       rollback-for="Exception" />
            <tx:method name="update*" propagation="REQUIRED"
                       rollback-for="Exception" />
            <tx:method name="delete*" propagation="REQUIRED"
                       rollback-for="Exception" />
            <tx:method name="*" propagation="SUPPORTS" />
        </tx:attributes>
    </tx:advice>
    <!-- 配置切面织入的范围,后边要把事务边界定在service层 -->
    <aop:config>
        <aop:advisor advice-ref="advice" 
                     pointcut="execution(* org.example.service.*(..))" />
    </aop:config>

    <!-- 配置SessionTemplate，已封装了繁琐的数据操作，提交事务都不用做了。 -->
    <bean id="sqlSessionTemplate" class="org.mybatis.spring.SqlSessionTemplate">
        <constructor-arg name="sqlSessionFactory" ref="sqlSessionFactory" />
    </bean>

    <!-- 自动扫描组件，要把controller去除，他们是在spring-mvc.xml中配置，如果不去除会影响事务管理。 -->
    <context:component-scan base-package="org.example">
        <context:exclude-filter type="annotation"
                 expression="org.springframework.web.bind.annotation.RestController" />
    </context:component-scan>

    <!-- 配置 转换器，对于在basePackage设置的包（包括子包）下的接口类，
         如果接口类的全类名在Mapper.xml文件中和定义过命名空间一致，
         将被转换成spring的BEAN，在调用的地方通过@Autowired方式将可以注入接口实例 -->

    <!--
        <bean class="org.mybatis.spring.mapper.MapperScannerConfigurer">
            <property name="sqlSessionFactory" ref="sqlSessionFactory" />

            &lt;!&ndash;这里包名留意要不要改&ndash;&gt;
            <property name="basePackage" value="org.example.repository" />
        </bean>
    -->
</beans>

Spring整合Mybatis框架 [JavaConfig+注解]

这里演示的是最简单的功能配置，如果需要更多功能的支持，参考官方文档配置

下列是Mybatis的JavaConfig文件

程序清单：MybatisConfigurator.java

@Configuration
@EnableTransactionManagement
public class MybatisConfigurator {

    @Bean
    public static PropertySourcesPlaceholderConfigurer propertyPlaceholderConfigurer() {
        return new PropertySourcesPlaceholderConfigurer();
    }

    /**
     * 配置数据源
     *
     * @return 数据源
     */
    @Bean(name = "dataSource")
    public DataSource dataSource() {
        DruidDataSource dataSource = new DruidDataSource();
        dataSource.setDriverClassName("com.mysql.jdbc.Driver"); 
        dataSource.setUrl("jdbc:mysql://127.0.0.1:3306/ssm?   
                          useUnicode=true&characterEncoding=utf-8");
        dataSource.setUsername("root");
        dataSource.setPassword("root");
        return dataSource;
    }

    /**
     * 1. 创建SqlSessionFactory对象，
     * 2. 将依赖的数据源对象注入到SqlSessionFactory对象中
     * 3. SqlSessionFactory对象注入到spring容器中
     *
     * @param dataSource
     * @return
     * @throws Exception
     */
    @Bean("sqlSessionFactory")
    public SqlSessionFactory sqlSessionFactory(DataSource dataSource) throws Exception {
        SqlSessionFactoryBean sqlSessionFactoryBean = new SqlSessionFactoryBean();
        sqlSessionFactoryBean.setDataSource(dataSource);
        // 第一种方式通过读取配置文件
        sqlSessionFactoryBean.setConfigLocation(new ClassPathResource("mybatis-config.xml"));

        // 第二种方式通过代码设置
        // ResourcePatternResolver resourcePatternResolver = new                                                PathMatchingResourcePatternResolver();
        // sqlSessionFactoryBean.setMapperLocations(
        //     resourcePatternResolver.getResources("classpath:mapper/*Repository.xml"));
        // 别名，让*Repository.xml实体类映射可以不加上具体包名
        //sqlSessionFactoryBean.setTypeAliasesPackage("org.example.bean");
        return sqlSessionFactoryBean.getObject();
    }


    /**
     * 创建 Mapper接口的代理对象
     * <p>
     * 设置扫描数据访问层的范围
     * 1. xml方式
     *
     * @MapperScan <bean class="org.mybatis.spring.mapper.MapperScannerConfigurer">
     * <property name="sqlSessionFactoryBeanName"
     * value="sqlSessionFactory" />
     * <property name="basePackage" value="org.maoge.xmlssmdemo.dao" />
     * </bean>
     * 2. 注解方式
     * @MapperScan(basePackages = "org.example.repository")
     * 3. java config方式
     */
    @Bean
    public MapperScannerConfigurer initMapperScannerConfigurer() {
        MapperScannerConfigurer mapperScannerConfigurer = new MapperScannerConfigurer();
        mapperScannerConfigurer.setSqlSessionFactoryBeanName("sqlSessionFactory");
        // 设置扫描数据访问层接口的范围
        mapperScannerConfigurer.setBasePackage("org.example.repository");
        return mapperScannerConfigurer;
    }

    /**
     * 创建事务管理器，将数据源注入到事务管理器中
     * <p>
     * 1.xml方式
     * <bean id="transactionManager" class=
     * "org.springframework.jdbc.datasource.DataSourceTransactionManager">
     * <property name="dataSource" ref="dataSource" />
     * </bean>
     * 2.java config方式
     */
    @Bean("dataSourceTransactionManager")
    public DataSourceTransactionManager dataSourceTransactionManager(DataSource dataSource) {
        DataSourceTransactionManager dataSourceTransactionManager = new DataSourceTransactionManager();
        dataSourceTransactionManager.setDataSource(dataSource);
        return dataSourceTransactionManager;
    }

}

# 自定义切面

程序清单：LogAspect.java

@Aspect
@Component
public class LogAspect {

    private Logger logger = LoggerFactory.getLogger(getClass());

    /**
     * @Aspect 切面注解
     * @Component 开启注解扫描时，会被注入到Spring容器中
     *
     * 切点
     * - 对service层
     *
     * - 对controller层，需要使用cglib动态代理
     */
    @Pointcut(value = "execution(* org.example.service.*.*(..))")
    public void pointcut() {

    }

    @Before("pointcut()")
    public void before() {
        logger.info("before ...");
    }

    @After("pointcut()")
    public void after() {
        logger.info("after ...");
    }

    @AfterReturning("pointcut()")
    public void afterReturning() {
        logger.info("after return ...");
    }

    @Around("pointcut()")
    public Object around(ProceedingJoinPoint point) throws Throwable {
        logger.info("已经记录下操作日志@Around 方法执行前");
        Object obj = point.proceed();
        logger.info("已经记录下操作日志@Around 方法执行后");
        return obj;
    }

    @AfterThrowing(pointcut = "pointcut()", throwing = "e")
    public void afterThrowing(Exception e) {
        logger.info("after throw ...{}",e.getMessage());
    }
}

# aop:aspect和aop:advisor

在配置切面的时候，有两个标签有点不清楚，它们分别是<aop:aspect>和<aop:advisor>

区别:

  • 实现方式不同，对于定义<aop:aspect>而言，仅仅需要实现切面类即可，但是定义< aop:advisor>中引用的通知时，通知必须实现Advice接口，底层实现是相同的。

  • 使用场景不同，<aop:advisor>大多用于事务管理，<aop:aspect>大多用于日志，缓存。
