把数组排成最小的数

Gen 2020-10-16 剑指offer

# 题目描述

题目：输入一个正整数数组，把数组里所有数字拼接起来排成一个数，打印能拼接出的所有数字中最小的一个。例如输入数组{3,32,321}，则打印出这3个数字能排成的最小数字321323。

# 题目意图

这道题其实是希望我们能找到一个排序规则，数组根据这个规则排序之后能排成一个最小的数字。要确定排序规则，就要比较两个数字，也就是给出两个数字m和n，我们需要确定一个规则判断m和n哪个应该排在前面，而不是仅仅比较这两个数字的值哪个更大。

--------参考《剑指Offer》

这题考察了两个点

  • 如何处理大数问题
  • 如何寻找比较规则，以及有效规则的普适性

# 解题思路

一个非常直观的解决大数问题的方法就是把数字转换成字符串。另外，由于把数字m和n拼接起来得到mn和nm，它们的位数肯定是相同的，因此比较它们的大小只需要按照字符串大小的比较规则就可以了

为什么字符串的比较大小能够代表两个数字拼接后的比较，对String类源码分析

/** The value is used for character storage. */
private final char value[];

public int compareTo(String anotherString) {
    int len1 = value.length;
    int len2 = anotherString.value.length;
    int lim = Math.min(len1, len2);
    char v1[] = value;
    char v2[] = anotherString.value;

    int k = 0;
    while (k < lim) {
        char c1 = v1[k];
        char c2 = v2[k];
        if (c1 != c2) {
            return c1 - c2;
        }
        k++;
    }
    return len1 - len2;
}

String类是一个最终类，内部通过一个不可变字符数组存储字符串信息。

字符可以像数字一样比较，位数相同时，从最高位开始一一比较。为了使拼接的数字最小，那么最高位也最小，所以应该采用升序比较

解决这个问题，需要以下三个步骤：

  • int数组转换为String数组
  • 自定义一个排序规则，对String数组从小到大排序（保证拼接的数字是最小的数字）
  • 对排序后的字符串数组进行连接

# 测试用例

  • 功能测试（输入的数组中有多个数字，输入的数组中的数字有重复的数位，输入的数字只有一个数字）

  • 特殊输入测试（表示数组的指针为NULL指针）

# 对比较规则普适性的数学证明（了解）

证明之前定义的比较两个数字大小的规则是有效的。一个有效的比较规则需要3个条件：自反性、对称性和传递性。我们分别予以证明。

（1）自反性：显然有aa=aa，所以a等于a。

（2）对称性：如果a小于b，则ab<ba，所以ba>ab，因此b大于a。

（3）传递性：如果a小于b，则ab<ba。

# 代码实现

public String toBeMinNumber(int [] numbers) {
    if (numbers == null || numbers.length == 0) {
        return "";
    }
    String[] numStrs = new String[numbers.length];
    for (int i=0,len = numbers.length; i < len; i ++) {
        numStrs[i] = Integer.toString(numbers[i]);
    }
    Collections.sort(Arrays.asList(numStrs),(str1,str2) -> {
            String result01First = str1 + str2;
            String resultO2First = str2 + str1;
            return result01First.compareTo(resultO2First);
        }
    );
    StringBuilder stringBuilder = new StringBuilder();
    for (String str: numStrs) {
        stringBuilder.append(str);
    }
    return stringBuilder.toString();
}

# 总结

有效的解答，往往是简洁的，有效的解法它思考的维度是通过拼接后的字符串比较来描述描述两个数字字符串的大小，其实并不是两个数字的真实大小，而是两个数字字符谁在前面排列使结果是最小的。

我最先开始找的比较规则是关注与两个数字之间的，没有想到在前期把数字转换成字符数组，以结果为导向去判断两个数字谁小，这里的小，不是说数字的值小，而谁放在前面使拼接的数字最小，这就是比较规则。

反思一下自己在解决问题时，容易出现的思维误区：

  • 对数字上的每个数字进行比较，而不是两个拼接后的字符串之间的比较。

    这其实是整体与部分的差别，那么以后思考问题的时候，就应该注意如果对太细节的问题进行考虑，情况比较复杂的话，可以尝试从整体结果上进行思考。

  • 字符串是可以比较的，这一点没有深刻的认识到，之前在项目中也遇到过，但是没有记录下来。

  • 解决大数问题，一个通用的方法就是将它转化为字符数组。
