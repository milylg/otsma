Spring三种装配方式

Gen 2020-10-03 Spring

# 1. xml配置文件装配

  • bean的构造方法注入

  • bean的set方法注入

# 2. 自动装配: 使用Annotation注解标记相应的类

  • @Component(value="bean名称") 注解标记的泛化
    • @Repository(value="bean名称")用于标识数据访问层
    • @Service(value="bean名称")用于标识业务层
    • @Controller(value="bean名称")用于标识控制层
  • @Resource(name="bean名称") Java提供的默认按Bean实例的名称装配
    • (name="bean名称")
    • (type="bean类型")
  • @AutoWired按Bean类型装配，用于对bean的属性，set方法，构造方法进行注入，与注解处理器配合使用
  • @Qualifier(value="bean名称") 按Bean实例的名称装配，和@AutoWired配合使用

# 3. 基于JavaConfig配置装配Bean

使用注解自动扫描

  • @ComponentScan指定扫描的范围，相当于context:component-scan标签

    • 如果注解内没有参数，则表示扫描JavaConfig文件所在包以及子包内所有带注解的Bean
    • 能够在Spring中启用组件扫描

  • @Bean注解

# Autowire和Resource注解的区别?

  • 来源不同

    • autowire来自spring模块
    • resource来自jdk

  • 作用不同

    • autowire根据bean的类型注入,如果需要按照bean的名称则需要与Qualifier搭配使用
    • resource默认根据bean名称注入,也可通过配置根据bean类型注入.当按名称寻找不到,则按类型寻找,如果找不到抛异常

  • required

    • 如果没有匹配的bean，那么在应用上下文创建的时候，Spring会抛出一个异常。为了避免异常的出现，可以将@Autowired的required属性设置为false
    • 设置为false时，Spring会尝试执行自动装配，但是如果没有匹配的bean的话，Spring将会让这个bean处于未装配的状态

  • @Autowire注解的替换方案

# 组件扫描和自动装配不适用的场景该如何？

有时候自动化配置的方案行不通，因此需要明确配置Spring，有两种可选方案：JavaConfig和XML

# JavaConfig的特点

  • JavaConfig是更好的方案，因为它更为强大、类型安全并且对重构友好
  • 不应该包含任何业务逻辑
  • 不应该侵入到业务逻辑代码之中
  • 通常会将JavaConfig放到单独的包中，使它与其他的应用程序逻辑分离开

# 如何实践：配置Config类？

  • @Configuration配置注解，在类的层面上标注，表明它是一个配置类
  • @ComponentScan自动扫描，允许spring扫描
  • @Bean，在创建配置类的方法上标注，获取bean的名称与方法名相同，也可通过@Bean的name属性自定义

前提是开启组件扫描配置

<context:component-scan base-package="com.spring, com.config"/>
