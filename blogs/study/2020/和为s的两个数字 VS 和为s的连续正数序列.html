和为s的两个数字 VS 和为s的连续正数序列

Genlg 2020-10-18 剑指offer

# 题目描述

题目一：输入一个递增排序的数组和一个数字s，在数组中查找两个数，使得它们的和正好是s。如果有多对数字的和等于s，输出任意一对即可。

例如输入数组{1、2、4、7、11、15}和数字15。由于4＋11＝15，因此输出4和11。

# 题目分析

首先，题目明确了序列是有序的，且是从小到大排列，查找的数是两个，使它们的和为s。使用两层for循环固然能够解决问题，但是时间复杂度为O(N^2)，在这个朴素的算法中，出现了很多无意义的比较，那么减少比较的次数就能使算法得到优化。

在数组的首尾各放一个索引标记，如果两个索引的位置上的数相加大于s，且数组是递增的，那么尾部索引应该向前移动一位；如果相加的结果值小于s，就让首部的索引向后移动一位，直到相等为止（循环终止条件）。

上述的流程中，为什么相加大于s时，不能让首部向前移动呢，因为程序不能够判断尾部的数是否已经大于s，如果大于s，无论首部如何向前移动，也不能使结果等于s，且这样的做法无异于使算法流程变得复杂起来。要保证两个索引标记反向而行，确定循环操作的结束条件（无论有没有结果），能够极大简化流程。小于s时，亦然。

# 代码实现

public void printTwoNumber(int[] array, int target) {
    if (array == null || array.length == 0) {
        return ;
    }
    // 初始化索引标记位置
    int start = 0, end = array.length - 1;
    // 循环操作 - 判断相加结果是否等于目标值
    while (start < end) {
        int sum = array[start] + array[end];
        if (sum < target) {
            start ++;
        } else if (sum > target) {
            end --;
        } else {
            System.out.println("No.1 = " + array[start] + "  No.2 = " + array[end]);
            break;
        }
    }
}

# 扩展题

输入一个正数s，打印出所有和为s的连续正数序列（至少含有两个数）。例如输入15，由于1＋2＋3＋4＋5＝4＋5＋6＝7＋8＝15，所以结果打印出3个连续序列1～5、4～6和7～8。

# 问题分析

打印的是一个连续的正数序列，在某个范围内数的和等于目标值，从上题的两个值的和扩展到了一个范围的和，其实思路是一样的，只是比较的方式有一些不同。

# 代码实现

public void printOrderListBySum(int s) {
    if (s <= 1) {
        return;
    }
    int middle = s / 2;
    int min = 1, max = 1;
    while (min <= middle) {
        int compareResult = compareTo(min, max, s);
        if (compareResult > 0) {
            min ++;
        } else if (compareResult < 0){
            max ++;
        } else {
            printRangeBetween(min, max);
            max ++;
        }
    }
}

private void printRangeBetween(int min, int max) {
    for (int j = min; j <= max; j ++) {
        System.out.print(" " + j);
    }
    System.out.println();
}

private int compareTo(int min, int max, int s) {
    int sum = 0;
    for (int i = min; i <= max; i ++) {
        sum += i;
    }
    return sum - s;
}
