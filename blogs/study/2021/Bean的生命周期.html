Bean的生命周期（了解）

Genlg

# Bean的生命周期（了解）

Spring的生命周期可以分为4个阶段：

  1. 实例化
  2. 属性注入
  3. 初始化
  4. 销毁

# 实例化

通过反射获取类的定义信息，进行对象实例化

# 依赖注入

在容器中，将对象中使用注解标注的成员变量的对象引用通过构造方法或set方法注入进来

实例化bean的过程中，需要解决bean之间的依赖问题，引入了依赖注入(DI)这种技术

Tip：

所有Aware接口的作用：属性注入

# BeanNameAware接口

对象实现该接口，可以在容器初始化后获取对象在spring容器中的id名称

# BeanFactoryAware接口

在容器加载的时候，通过回调该接口的方法传入beanfactory实例给实现了该接口的对象

作用：让Bean获取配置他们的BeanFactory的引用

最常用实现是XmlBeanFactory

特性 BeanFactory ApplicationContext
Bean 实例化/装配 Yes Yes
自动 BeanPostProcessor 注册 No Yes
自动 BeanFactoryPostProcessor 注册 No Yes
便捷的 MessageSource 访问( i18n) No Yes
ApplicationEvent 发送 No Yes

Tip:

不推荐使用，因为结果是进一步加大Bean与Spring的耦合，所以能通过DI注入进来的尽量通过DI来注入

# ApplicationContextAware接口

从已有的spring上下文取得已实例化的bean，获取spring上下文对象。

在一些自定义的类中，不能够通过属性注入的方式注入，可以通过对实现该接口的类进行封装获取容器中的对象。

# BeanPostProcessor：后置处理器

作用：bean的自定义初始化方法前后进行处理工作

作用域：全局，容器中的所有的对象

bean指定组建的init方法和destroy的三种方法 1：在配置类中 @Bean(initMethod = "init",destroyMethod = "destory")注解指定 2：实现InitializingBean接口重写其afterPropertiesSet方法，实现DisposableBean接口重写destroy方法 3：利用java的JSR250规范中的@PostConstruct标注在init方法上，@PreDestroy标注在destroy注解上

案例演示：

[main] INFO  org.example.bean.Account - bean 初始化
[main] INFO  org.example.bean.AccountBeanPostProcessor - account PostProcess Before Initialization...
[main] INFO  org.example.bean.Account - custom init method
[main] INFO  org.example.bean.AccountBeanPostProcessor - account PostProcess After Initialization...

# InitializingBean接口

为bean提供了初始化方法的方式

与init-method方法的功能相同

spring为bean提供了两种初始化bean的方式，实现InitializingBean接口，实现afterPropertiesSet方法，或者在配置文件中同过init-method指定。两种方式可以同时使用，如果调用afterPropertiesSet方法时出错，则不调用init-method指定的方法。

# DisposableBean接口

在容器关闭前，调用对象的自定义销毁destory方法

作用域：单例对象

与 destroy-method方法的功能相同
